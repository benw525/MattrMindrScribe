# Auphonic Audio Cleaning — Pipeline Integration

Preprocesses legal audio recordings through Auphonic's API before feeding them
into the Whisper → AssemblyAI → Claude transcription pipeline.

## Architecture

```
User uploads audio
        │
        ▼
┌─────────────────┐   BullMQ job: auphonic-clean
│  Auphonic API   │◄──────────────────────────────
│  (Speech Isol.) │   POST /api/simple/productions
└────────┬────────┘
         │ async processing...
         │
         ▼ webhook POST /webhooks/auphonic
┌─────────────────┐
│  Download clean  │
│  WAV file        │
└────────┬────────┘
         │ BullMQ job: whisper-transcribe
         ▼
┌─────────────────┐
│  Whisper (STT)  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  AssemblyAI     │  Speaker diarization
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Claude Opus    │  Speaker ID + refinement
└─────────────────┘
```

## Files

| File | Purpose |
|---|---|
| `auphonic-service.js` | Core API client — preset creation, file upload, status, download |
| `auphonic-webhook.js` | Express route that receives Auphonic completion callbacks |
| `pipeline-integration.js` | BullMQ wiring — queues, workers, entry point |
| `.env.example` | Required environment variables |

## Setup

### 1. Get your Auphonic API key

Sign up at https://auphonic.com, then go to:
https://auphonic.com/accounts/api_key/

### 2. Install dependencies

```bash
npm install axios form-data bullmq express
```

### 3. Configure environment

```bash
cp .env.example .env
# Fill in AUPHONIC_API_KEY and AUPHONIC_WEBHOOK_URL
```

### 4. Mount the webhook in your Express app

```javascript
const express = require('express');
const { mountWebhook, startTranscriptionPipeline } = require('./pipeline-integration');

const app = express();
mountWebhook(app);

// Example: trigger pipeline from an upload route
app.post('/api/matters/:id/upload', async (req, res) => {
  const matterId = req.params.id;
  const audioFilePath = req.file.path; // from multer or similar

  const jobId = await startTranscriptionPipeline(matterId, audioFilePath);
  res.json({ jobId, message: 'Transcription pipeline started' });
});

app.listen(3000);
```

### 5. First run — preset creation

On the first submission, the service auto-creates an Auphonic preset
configured for legal transcription. The UUID is logged to console —
save it to `AUPHONIC_PRESET_UUID` in your `.env` so it's reused.

## Preset Configuration (what it does to your audio)

| Algorithm | Setting | Why |
|---|---|---|
| Speech Isolation | 100% noise removal | Strips everything except human voice |
| Remove Reverb | 80% | Cuts room echo but keeps voice natural |
| Remove Breaths | 50% | Moderate — avoids unnatural gaps in speech |
| Voice AutoEQ | Enabled | Normalizes frequency across different speakers/mics |
| Adaptive Leveler | Moderate | Balances volume between speakers |
| High-Pass Filter | Enabled | Removes low-frequency hum and interference |
| Loudness Target | -16 LUFS | Standard speech loudness |
| Output Format | WAV | Lossless — no quality loss before Whisper |

## Webhook vs. Polling

The integration is webhook-driven by default. Auphonic POSTs to your
`/webhooks/auphonic` endpoint when processing completes. This is the
recommended approach.

A polling fallback (`waitForCompletion()`) exists in the service for
cases where webhook delivery fails or for local development without
a public URL. Don't use it in production if you can avoid it.

## Pricing Considerations

Auphonic charges per audio hour processed. The free tier includes
2 hours/month. For production legal workloads, you'll need a paid plan.
Check https://auphonic.com/pricing for current tiers.

## Adjusting Denoise Aggressiveness

If Speech Isolation is too aggressive for certain recordings (e.g.,
removing parts of quiet speech), you can override per-submission:

```javascript
await auphonicService.submitForCleaning(filePath, matterId, {
  denoiseMethod: 'dynamic_denoise', // Keeps music + speech, removes noise only
});
```

Available methods:
- `speech_isolation` — Most aggressive. Voice only. Best for depositions/hearings.
- `dynamic_denoise` — Keeps speech + music, removes all other noise.
- `static_denoise` — Removes only static/broadband noise. Gentlest.
